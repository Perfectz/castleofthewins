# Before / After Comparison

## 1. Adventure Menu Entry Points

- Before: `Mission`, `Journal & Log`, and `How to Play` each implied a separate support surface.
- After: `Mission`, `Field Guide`, and `Guide` now act as deep links into one shared support architecture.
- Why it matters: the menu communicates the system structure more honestly. The player sees one support surface with multiple entry intents, not three disconnected reference modals.

Evidence:
- `screenshots/before/before-mobile-01-run-menu.png`
- `screenshots/after/after-mobile-01-run-menu.png`
- `screenshots/before/before-desktop-01-run-menu.png`
- `screenshots/after/after-desktop-01-run-menu.png`

## 2. Journal Home to Field Guide Current

- Before: `Run Journal` behaved like an archive-heavy scroll with mixed story, objective, town, and telemetry content.
- After: `Field Guide` defaults to `Current`, with a clear summary row and section chips that expose the support architecture directly.
- Why it matters: the default support surface now starts with the current run instead of making the player infer structure from a long stack of sections.

Evidence:
- `screenshots/before/before-mobile-02-journal-current.png`
- `screenshots/after/after-mobile-02-journal-current.png`
- `screenshots/before/before-desktop-02-journal-current.png`
- `screenshots/after/after-desktop-02-journal-current.png`

## 3. Mission Flow

- Before: `Mission` launched a dedicated `Mission Briefing` modal.
- After: `Mission` lands directly on the `Mission` section inside `Field Guide`.
- Why it matters: briefing becomes part of the shared support model instead of a parallel dead-end screen.

## 4. Guide Flow

- Before: `How to Play` launched a separate rules modal.
- After: `Guide` lands on the `Guide` section inside `Field Guide`.
- Why it matters: rules reference now lives inside the same support surface as mission and run history, which reduces the number of conceptually overlapping modals.

## 5. Chronicle Structure

- Before: chronicle-style content was buried late in the journal scroll.
- After: `Chronicle` is an explicit section inside `Field Guide`.
- Why it matters: archive content is still available, but it no longer dominates the default support view.
