# Comparison

The raw baseline captures are necessary, but they are not efficient to study in sequence. The after-side artifacts add structure, ordering, and annotation so a reviewer can compare the system as a whole instead of opening dozens of PNGs one by one.

What the organized after artifacts add:

- `screenshots/after/mobile-baseline-atlas-board.png` turns the most important phone-sized captures into a single contact sheet with stable labels and route notes.
- `screenshots/after/desktop-baseline-atlas-board.png` creates the same study view for desktop so responsive differences are visible at a glance.
- `screenshots/after/navigation-map-board.png` converts the baseline into a flow narrative, which helps distinguish entry, town loop, run utility, progression, and fail-state surfaces.
- `screenshots/after/mobile-hud-callout-board.png` annotates one representative HUD frame so the relationship between board, minimap, stat stack, feed, and controls becomes explicit.

Compared with the raw screenshots, the after-side artifacts reduce three kinds of friction:

- Discovery friction: reviewers no longer need to remember filenames or open each state manually.
- Interpretation friction: the boards make naming, grouping, and route context visible.
- Publishing friction: the boards can be dropped into docs, discussions, and blog drafts without additional layout work.
