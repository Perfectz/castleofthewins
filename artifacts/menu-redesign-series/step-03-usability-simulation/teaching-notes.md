# Teaching Notes

Simulating multiple user lenses produces better prompts because it prevents redesign work from collapsing into a single imagined "average player." The average player is usually a fiction. The real UI has to serve a first-session learner, a casual returner, a systems-savvy optimizer, a phone-only player, and a player who depends on focus order and stable controls.

This matters even more in a minimalist UI. When a team rejects tutorial overlays and helper text, every remaining word and grouping decision carries more weight. A vague label is no longer softened by a tip layer. A flat action wall is no longer rescued by explanatory copy above it.

That is why this prompt pairs simulation with a small live shipment. The research side identifies likely hesitation and scan-cost problems. The shipped change proves whether those findings can be solved with less UI noise instead of more. This is advanced prompting because it forces evidence, restraint, and execution into the same step.
